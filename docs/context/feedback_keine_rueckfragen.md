Nicht fragen, sondern machen, wenn der nächste Schritt klar ist (z.B. Bug fixen + weitermachen).

**Why:** Rückfragen bremsen den Workflow und sind unnötig wenn die Richtung eindeutig ist.
**How to apply:** Wenn ein Bug auftritt: fixen. Wenn ein Batch fertig ist und weitere Arbeit ansteht: weitermachen. Nur fragen wenn echte Entscheidungen anstehen.

Zusätzlich: Keine optimistischen Projektionen auf Basis von Einzeltests. Ein Benchmark auf 1 Artikel ist keine Validierung für 3000. Immer klar sagen wenn Evidenz dünn ist. Bei Modell- oder Prompt-Änderungen immer Testlauf auf ≥20 bekannten Artikeln machen bevor ein voller Run startet.
